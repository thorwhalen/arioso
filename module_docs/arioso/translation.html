arioso.translation

Parameter translation layer.

Translates between the 40 unified affordance names and each platform’s native parameter names, applying type coercions along the way.

arioso.translation.make_generate_func(config: dict) → Callable[source]

Build a generate() callable from a platform config.

This is the fallback when no custom adapter.py is provided. For REST API platforms, it builds the HTTP function from the config and wraps it with parameter translation and output normalization.

Parameters:

config – A PLATFORM_CONFIG dict.

Returns:

A callable that accepts unified affordance kwargs and returns a Song.

arioso.translation.make_kwargs_trans(param_map: dict, *, on_unsupported: str = 'warn') → Callable[[dict], dict][source]

Build a kwargs translation function from a platform’s param_map.

The returned function transforms:

{common_name: value, ...}  ->  {native_name: coerced_value, ...}
Parameters:

  • param_map – Dict mapping common affordance names to native config dicts. Each value has keys: native_name (str), coerce (optional callable), adapter_handled (optional bool).

  • on_unsupported – What to do with params not in param_map. One of ‘warn’, ‘raise’, or ‘ignore’.

Returns:

A function that translates outer kwargs to inner kwargs.
